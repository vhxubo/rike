# Rike 第一阶段实现计划

## 目标

依据 `docs/superpowers/specs/2026-07-14-rike-foundation-design.md`，完成可运行的 React 基础工程、分层目录、基础组件、组件陈列首页、测试与交接文档。

## 实现步骤

1. 使用 fnm 切换到 Node 24，建立 Vite、React、TypeScript、Tailwind CSS、ESLint 和 Vitest 配置。
2. 使用 pnpm 安装并锁定 React、Motion、Zustand、localForage、`tailwind-variants` 和 PWA 等依赖。
3. 定义纸本日课设计令牌、全局样式、响应式容器和降低动态效果规则。
4. 实现 UI、纸本、布局和反馈组件。
5. 定义 calendar、backup 和 daily-guidance 的类型契约。
6. 实现带日期和示例每日一句的组件陈列首页。
7. 添加关键组件和页面测试。
8. 执行 `typecheck`、`lint`、`test` 和 `build`。
9. 编写 `docs/handoffs/` 交接记录并更新索引。

## 视觉计划

- 工作台：`#E8ECE7`，用于区分应用背景与纸页。
- 宣纸：`#FFFDF7`，只用于主要内容表面。
- 墨色：`#20231F`，用于正文与主边线。
- 朱砂：`#B74736`，用于日期与重点动作。
- 青绿：`#3E7665`，用于完成和辅助状态。
- 石墨：`#6C706B`，用于次要文字。
- 标题：系统宋体栈；正文：系统黑体栈；数据标签：系统等宽字体栈。
- 标志元素：左侧朱砂装订线、横线纸节奏和紧凑日期印记。

## 验收

- 页面在 360px 手机宽度和常见桌面宽度无横向溢出。
- 所有交互控件有键盘聚焦状态和可访问名称。
- 动效在减少动态效果设置下自动降级。
- 领域契约不依赖具体存储实现。
- 四项工程命令全部通过。
