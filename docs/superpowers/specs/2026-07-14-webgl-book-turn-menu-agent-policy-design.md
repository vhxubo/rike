# Rike 真实书页翻动、菜单视图与 Agent 限制设计

## 1. 背景

当前 header 右侧直接展示日、周、月按钮，工具栏菜单只包含周视图和月视图。返回快照存在时，左上角工具栏按钮被返回按钮完全替代，因此子页面无法直接打开视图菜单。

当前翻页使用 DOM 平移、轻微 3D 倾斜和 CSS 局部折痕。折痕位置已经跟随手指，但纸页本身仍是平面，阴影仅由渐变层构成，无法满足真实书本中固定书脊、纸张卷曲、正反面与动态光照的一致性。

本次将日、周、月入口全部收回工具栏菜单，修正今日根页面返回按钮，并使用 DOM 与 WebGL Canvas 混合方案重建真实书页翻动。同时在仓库根目录增加强制 Agent 限制：用户未明确要求时不得编写、修改或运行测试与验收。

## 2. 目标

- header 右侧不再显示日、周、月切换。
- 工具栏菜单完整提供日视图、周视图和月视图。
- 子页面同时提供返回和菜单入口。
- 今天的日视图是根页面，左上角不显示返回按钮。
- 使用左侧固定书脊、可弯曲网格和真实正反方向的 WebGL 书页翻动。
- 页面静止时继续使用真实 DOM，保留表单、文本和无障碍能力。
- 建立仓库级禁止默认测试与验收的 Agent 规则。

## 3. 非目标

- 不把整个应用永久迁移到 Canvas。
- 不使用 Three.js 或其他完整 3D 引擎。
- 不改变日、周、月数据、统计、主题或 FAB 业务规则。
- 不把视图层级切换表现成书页翻动。
- 本轮不新增、修改或运行测试，不执行任何验收。

## 4. header 与菜单

### 4.1 header

- header 右侧只保留“查看本周总结”按钮。
- 删除右侧日、周、月按钮组及相关 props。
- header 左侧始终保留菜单入口。
- 有返回上下文时，左侧依次显示返回按钮和菜单按钮。
- 没有返回上下文时只显示菜单按钮。

### 4.2 菜单项目

菜单功能区依次提供：

1. 日视图
2. 周视图
3. 月视图
4. 本周总结
5. 总统计

主题分隔区继续提供系统、日间和夜间。

- 日、周、月菜单项使用当前页面锚点进入对应日历视图。
- 普通日历页面通过菜单切换视图时不制造额外返回层级。
- 从周总结或统计页进入日历视图时保留进入子页面之前的返回快照。
- 菜单关闭、Escape、外部点击和焦点恢复规则保持不变。

## 5. 今日根页面与返回

定义根页面：

```text
workspacePage === calendar
calendarView === day
selectedDate === today
```

- 一旦进入根页面，立即清除过时的返回快照。
- 左上角只显示菜单，不显示返回。
- 从统计或总结页通过菜单进入今天日视图时，也清除返回快照。
- 从历史日期、周视图或月视图进入日视图时，如果日期不是今天，保留已有返回快照。
- FAB 回到今天并落在日视图时同样触发根页面清理。

## 6. 混合渲染架构

### 6.1 DOM 与 Canvas 分工

- 静止状态使用真实 DOM 页面。
- WebGL Canvas 只在同层级日期范围翻动期间显示。
- Canvas 位于固定 header 下、目标页 DOM 上方。
- 第一帧 WebGL 成功绘制后才隐藏活动 DOM 页，避免空白闪烁。
- 翻页完成或回弹后销毁临时 Canvas 展示状态，恢复对应真实 DOM。
- 输入、按钮和无障碍树始终以 DOM 为最终来源。

### 6.2 模块边界

- `PageTurn`：手势识别、方向、完成/回弹判定和日期提交。
- `PageTextureCache`：DOM 捕获、纹理键、失效、预取和释放。
- `BookPageRenderer`：WebGL context、网格、shader、纹理上传和绘制。
- `book-page-geometry`：无 DOM 依赖的书脊约束、拖动点投影、卷曲半径和网格位置计算。
- `book-page-shaders`：顶点变形、正反面采样、法线光照和阴影。

## 7. 页面纹理

### 7.1 捕获

- 使用 `html2canvas` 捕获可翻动纸页，不捕获固定 header 和 FAB。
- 按设备像素比绘制，但限制最大纹理尺寸，避免移动端显存过高。
- 捕获完成后转换为 WebGL texture，原始临时 canvas 在不再需要时释放。
- 表单当前值、纸张横线、主题颜色和状态样式必须反映捕获时 DOM。

### 7.2 缓存键

纹理键至少包含：

- 日期或范围标识
- 日/周/月视图
- 当前主题
- 纸页宽高
- 设备像素比
- 计划记录版本或内容修订号

只保留当前页、上一页和下一页三张纹理。日期、内容、主题、视口或 DPR 变化时使相关缓存失效。

### 7.3 预取

- 页面稳定后通过 `requestIdleCallback` 预取当前页与相邻页。
- 不支持 `requestIdleCallback` 时使用短延迟调度。
- 页面内容正在输入时不反复捕获；输入停止后的防抖窗口再更新当前纹理。
- 用户开始翻页但纹理未就绪时，页面保持 DOM 静止并记录最新触点；纹理完成后从最新进度接管。

## 8. 真实书页几何

### 8.1 坐标与书脊

- 左侧装订线是固定书脊。
- 书页局部坐标以左上角为原点，宽和高归一化到 0 至 1。
- 网格左边界所有顶点固定在书脊平面，不能平移或拉伸。
- 网格使用约 32×20 分段，允许按设备性能下调。

### 8.2 拖动约束

- 手指位置转换为书页局部坐标。
- 拖动点投影到书脊上下端点共同允许的可达区域，防止页角距离超过原始纸张长度。
- 折痕轴通过拖动点和固定书脊约束计算，不使用固定垂直暗带。
- 卷曲半径随拖动距离变化：开始较大、翻到中段变小、跨过书脊后再次展开。
- 手指纵向位置改变折痕轴角度与受力页角，顶部、中部和底部拖动呈现不同形态。

### 8.3 下一页

- 当前页左边界固定。
- 右侧自由边缘被拖向左侧。
- 网格顶点围绕动态圆柱曲面卷曲，越过书脊后显示纸张背面。
- 下层目标页逐步显露但不做不真实的整体平移。

### 8.4 上一页

- 使用下一页几何状态的严格时间反演。
- 上一张纸从书脊左侧以背面朝向进入。
- 纸张卷过书脊后向右展开，最终成为完整前景页。
- 不允许把当前页简单向右平移或反向旋转来冒充上一页。

## 9. WebGL 光照与阴影

### 9.1 正反面

- 正面采样页面纹理。
- 背面使用宣纸底色、弱横线和低透明度正面透印。
- 背面纹理方向按真实纸张翻转关系处理，不能保持正向文字。

### 9.2 光照

- 顶点或片元 shader 根据网格法线计算柔和漫反射。
- 折痕凸面产生窄高光，凹面产生连续暗部。
- 光源固定在视口上前方，避免光影随页面方向无规律翻转。
- 日间与夜间使用主题语义颜色和不同曝光参数。

### 9.3 阴影

- 书脊附近产生接触阴影。
- 翘起纸页向目标页投射随高度与曲率变化的软阴影。
- 阴影宽度随纸页离底页距离扩大，透明度随距离降低。
- 翻页开始和结束时阴影归零。
- 删除现有 CSS `.page-turn-fold`、`.page-turn-edge` 与黑带渐变。

## 10. 状态机

WebGL 翻页状态：

```text
idle -> preparing -> dragging -> completing -> idle
                         \-> reverting -> idle
                         \-> failed -> fallback -> idle
```

- `preparing`：请求纹理和 WebGL 资源，DOM 保持显示。
- `dragging`：Canvas 接管视觉，持续使用最新触点绘制。
- `completing`：Motion 或 RAF 驱动几何进度到完成状态。
- `reverting`：几何严格沿当前路径返回初始状态。
- `failed`：清理资源并进入简化平移降级。
- 动画令牌继续阻止取消后的旧回调提交日期。
- 双指缩放取得控制权时取消 Canvas 翻页并恢复 DOM。

## 11. 性能与资源

- WebGL Canvas 按实际纸页尺寸设置 CSS 尺寸，buffer 尺寸考虑 DPR 和最大纹理限制。
- 每帧只更新必要 uniform 或动态顶点，不重新捕获 DOM。
- 使用一个当前 renderer，不同时创建多个 WebGL context。
- 纹理、framebuffer、buffer、shader program 和 RAF 在卸载或失效时显式释放。
- 监听 `webglcontextlost` 和 `webglcontextrestored`。
- 页面隐藏时暂停 RAF，恢复时重新绘制。
- 低性能设备可降低网格分段与纹理分辨率，不改变几何规则。

## 12. 错误与降级

- `html2canvas` 捕获失败：取消当前真实书页动画，使用无阴影轻量平移。
- WebGL 不可用或 context lost：恢复 DOM，释放资源并降级。
- 纹理超过显存限制：降低分辨率重试一次，仍失败则降级。
- 捕获期间日期、内容或主题变化：废弃旧任务结果。
- `prefers-reduced-motion`：不捕获纹理、不创建 WebGL，直接提交日期范围。
- 降级动画不使用当前被否定的 CSS 折痕或阴影。

## 13. Agent 写入限制

仓库根目录新增 `AGENTS.md`，包含以下强制规则：

```text
除非用户明确要求测试或验收：
- 禁止新增或修改测试文件。
- 因当前产品变更而过时的测试文件必须移除。
- 禁止运行测试、类型检查、Lint、构建、预览、开发服务器、截图和人工验收。
- 禁止声称改动已通过验证。
```

- 用户只授权某一种验证时，仅可执行该种验证，不得扩大范围。
- 普通功能开发默认只完成代码与文档实现并标记“未验证”。
- 本规则适用于根目录及全部子目录。
- 本规则取代需求记录中此前“完整任务结束后必须自动运行最终验证”的旧规则。
