# Rike 固定头部、动态折痕与回到今天 FAB 设计

## 1. 背景

现有页面已经支持柔软纸页平移，但翻页容器仍使用 `overflow-hidden`，纸页和阴影被限制在内容框内；折痕始终贯穿固定位置，不能反映用户实际触点。页面壳同时存在顶部设置按钮、内容区日/周/月切换、区块英文标题和底部 TabBar，信息入口分散。

本次调整重新组织页面壳：固定 header 集中工具栏、周总结和日历视图切换；移除底部导航与装饰性区块标题；每日一句回到页面首行；翻页允许越出纸页框并按触点生成局部折痕；回到今天改为可拖动、位置持久化的 FAB。

## 2. 目标

- header 始终吸附在视口顶部，不参与滚动和翻页动画。
- 右上角集中本周总结与日/周/月切换，替代设置图标。
- 移除底部 TabBar、`TODAY` 和 `PLAN` 展示文字。
- 每日一句占据原 `TODAY` 首行，日期导航和计划内容紧随其后。
- 翻页纸张、阴影和折痕可以越出内容框，不被直接父级裁切。
- 折痕位置、斜率和光影跟随手指纵向位置。
- 回到今天使用可拖动 FAB，并持久化用户放置位置。
- 手机端纸页贴合屏幕左右边缘，不增加页面外侧 gutter。

## 3. 非目标

- 不移除工具栏中的统计和主题能力。
- 不改变日、周、月数据、统计口径或 v3 持久化迁移。
- 不让 header 或 FAB 参与日期翻页。
- 不使用 SVG 位移滤镜、Canvas 或 WebGL 生成折痕。
- 不移除内容自身为可读性保留的左右内边距。

## 4. 固定 header

### 4.1 结构

header 位于 `PageTurn` 外部，采用吸顶布局：

```text
左侧：工具栏/返回 + 日课
右侧：本周总结 + 日 / 周 / 月
```

- 普通页面左侧显示工具栏按钮；子页面显示统一返回按钮。
- 手机端隐藏 `Rike` 英文副标题，为右侧控件释放空间。
- 原设置图标完全移除。
- 本周总结使用紧凑图标按钮并提供可访问名称。
- 日、周、月使用三个等宽小按钮，暴露当前选中状态。
- 320px 宽度下所有控件必须在单行内完整显示，不产生横向滚动。

### 4.2 吸顶与层级

- header 使用 `position: sticky; top: 0`，滚动时始终位于视口顶部。
- header 使用纸面背景、底边线和适度透明度或背景模糊，内容滚动到其下方时仍可辨识。
- header 层级高于纸页、折痕和统计内容，低于已打开的工具栏浮层。
- header 不复制进相邻翻页目标，不随日期页面移动。
- 顶部安全区通过 `env(safe-area-inset-top)` 处理。

## 5. 页面内容重排

- 删除所有页面上的底部 TabBar 渲染和预留空间。
- 删除 `BottomNav` 组件及 `AppShell` 的 navigation 接口；组件陈列页同步适配。
- 删除日历页可见的 `TODAY` 和 `PLAN` eyebrow 文字。
- header 下第一行展示每日一句，保留引用图标和纸本边线，不显示额外英文标题。
- 日期或范围导航位于每日一句下方。
- 计划、周视图或月视图内容直接位于日期导航后方，不保留 `PLAN` 标题留白。
- 周总结与总统计页不显示装饰性英文 eyebrow，只保留实际中文页面标题和范围。
- 手机端 `PageContainer` 外侧左右 padding 为 0；`sm` 及以上保留现有响应式 gutter。
- `RuledSection` 等内容容器继续保留内部阅读 padding。

## 6. header 右侧操作

### 6.1 本周总结

- 本周总结按钮始终位于右上角，不再在日视图内容区显示重复按钮。
- 点击后进入当前选中日期所在周的总结，并建立原有返回快照。
- 在周总结或统计页面中仍显示该按钮；点击时刷新为当前锚点所在周总结。

### 6.2 日、周、月切换

- 移除内容区原有 `ViewSwitcher`。
- 日、周、月按钮始终位于右上角。
- 普通日历页面切换时不建立返回快照。
- 从周总结或统计页面点击视图按钮时，进入选中日期或当前锚点对应的日历视图，并保留统一返回能力。
- 日、周、月切换继续支持双指或触控板缩放入口。

## 7. 不裁切的纸页运动

- `PageTurn`、纸页及直接布局父级不使用 `overflow-hidden` 或 `overflow-clip` 裁切纸页。
- 纸页横向移动、轻微倾斜、边缘阴影和局部折痕可以越出纸页框与桌面端边线。
- 应用最外层负责避免产生可滚动的横向滚动条，但不能在纸页的直接父级截断视觉。
- 手机端页面贴满视口，纸页可以短暂越出视口边缘；桌面端可在外侧 gutter 中看到纸页和阴影。
- 目标页仍位于当前页下层并禁止交互，不因 overflow 调整破坏焦点或点击边界。

## 8. 跟随触点的局部折痕

### 8.1 触点采样

- 指针按下时记录纸页边界。
- 横向翻页意图成立后持续计算 `pointerY / pageHeight`，限制在 0 到 1。
- 按钮触发翻页使用默认纵向位置 `0.6`。
- 双指缩放或指针取消时清理触点折痕状态。

### 8.2 折痕视觉

- 折痕以触点纵向位置为中心，使用约 140–220px 高的局部渐变区域，不再贯穿整页。
- 靠近顶部时折痕向上偏斜，靠近底部时向下偏斜，中部接近水平。
- 折痕由纸色高光、窄暗线和外侧软投影组成，强度继续随翻页进度在中段达到峰值。
- 折痕中心跟随手指平滑移动，不使用上/中/下离散档位。
- 翻页结束或回弹完成时，折痕透明度、倾斜和位移全部归零。

### 8.3 运动函数

纯运动模型扩展输入：

```ts
getPageTurnVisual(progress, direction, pointerYRatio)
```

输出除现有平移、轻倾斜、视差和强度外，增加折痕中心百分比与折痕角度。函数在 `progress` 为 0 或 1 时不得残留折痕。

## 9. 可拖动回到今天 FAB

### 9.1 显隐与动作

- FAB 位于工作区上层，不参与页面滚动和翻页。
- 日视图仅非今天时显示。
- 周视图仅当前周不包含今天时显示。
- 月视图仅当前月不包含今天时显示。
- 周总结仅当前总结周不包含今天时显示。
- 总统计的周、月、年范围仅当前范围不包含今天时显示。
- 全部统计不显示 FAB。
- 点击或键盘激活执行当前页面语义下的回到今天，并在状态更新后隐藏。

### 9.2 拖动与点击区分

- FAB 使用 Pointer Events 和 pointer capture。
- 移动距离超过小阈值后进入拖动模式，松手只保存位置，不执行回到今天。
- 未超过阈值的轻点执行回到今天。
- 键盘用户通过正常按钮激活，不依赖拖动。
- 拖动期间禁止文字选择和浏览器原生拖拽。

### 9.3 位置持久化

- 默认位置位于右下角，考虑 `safe-area-inset-right` 和 `safe-area-inset-bottom`。
- 保存 FAB 中心相对于可视宽高的归一化坐标，不保存固定设备像素。
- 恢复、屏幕旋转和窗口缩放时重新换算并夹紧到可视区域。
- 夹紧范围同时避开固定 header 和屏幕边缘。
- 使用独立轻量存储键，不进入计划 store 或备份数据。
- 位置读取或写入失败时使用默认右下位置，不影响回到今天功能。

## 10. 组件边界

- `AppShell`：移除底部导航，只负责工作台和页面宽度。
- `StickyWorkspaceHeader`：组合左侧工具栏/返回、标题、本周总结和视图按钮。
- `PageTurn`：只组合可翻动页面，接收并更新触点纵向比例。
- `TodayFab`：独立处理显隐、拖动、夹紧、持久化和点击。
- `DailyPlanPage`：根据工作区页面计算 header 操作、FAB 显隐和回到今天动作。
- `RuledSection`：允许不传 eyebrow 时不产生标题占位。

## 11. 错误与降级

- FAB 存储不可用时保留当前会话位置，并在刷新后回退默认位置。
- FAB 恢复坐标非法时回退右下角。
- 无法获取纸页尺寸时，折痕使用默认 60% 纵向位置。
- `prefers-reduced-motion` 下不显示动态折痕或越界运动，直接提交日期范围。
- 不支持 `backdrop-filter` 时 header 使用不透明纸面背景。
- 页面卸载、指针取消、窗口失焦和双指接管时必须释放 pointer capture 与折痕状态。

## 12. 测试

- header 位于翻页容器外并具有吸顶样式。
- 右上角本周总结和日/周/月按钮正常工作。
- 原设置按钮、底部导航、内容区视图切换、`TODAY` 和 `PLAN` 不再出现。
- 每日一句位于页面首行，日期导航和计划内容顺序正确。
- 手机断点的页面外侧左右 padding 为 0。
- 纸页直接父级允许 overflow，桌面 gutter 不裁切折痕与阴影。
- 折痕中心和角度随顶部、中部、底部触点连续变化。
- 按钮翻页使用 60% 默认折痕位置。
- FAB 在日、周、月、周总结和统计范围中的显隐正确。
- FAB 轻点回到今天，拖动不触发点击。
- FAB 位置保存、恢复、视口变化夹紧和非法值回退。
- header 控件从子页面进入日历视图后，统一返回仍恢复原上下文。
- 既有缩放、统计、主题、v3 迁移和输入交互回归。

阶段完成时执行类型检查、Lint、全部测试和生产构建。真实手机需要人工检查 sticky header、安全区、局部折痕跟手和 FAB 拖动手感。

## 13. 完成标准

- 页面滚动时 header 始终可见且不参与翻页。
- 右上角集中周总结与日/周/月切换，设置和底部 TabBar 不再出现。
- `TODAY`、`PLAN` 被移除，每日一句成为内容首行。
- 手机端页面无外侧左右 gutter。
- 翻页视觉可以越出纸页框，并且折痕随手指纵向位置变化。
- 回到今天只以可拖动 FAB 形式出现，显隐、动作和位置恢复正确。
- 自动化验证全部通过，人工验收项记录到交接文档。
