# Rike 柔软翻页与月视图调整设计

## 1. 背景

第三阶段已经实现日、周、年视图和整页 CSS 3D 翻转。实际体验中，当前纸页围绕固定边缘旋转 180°，拖动距离与旋转角度线性对应，并使用固定窄阴影，整体更像硬质门板，翻页过程生硬。

本次调整以更柔软、克制的纸页运动替换原有整页翻转，同时将日历层级从日、周、年改为日、周、月。原有年视图和十二个月总览完全移除。

## 2. 目标

- 日历视图改为 `day / week / month`。
- 月视图一次展示一个月，并用相邻月份日期补齐完整周。
- 月视图中的所有日期都可点击并进入对应日计划页。
- 日、周、月同层级导航统一使用柔软纸页翻动。
- 保留左右跟手、完成、回弹、按钮自动翻页和减少动态效果降级。
- 从现有 v2 本地状态无损迁移到 v3。

## 3. 非目标

- 不保留年视图入口或十二个月总览。
- 不使用 Canvas、WebGL 或物理网格模拟真正的纸张弯曲。
- 不改变计划模板、编辑权限、状态派生和统计规则。
- 不改变原生缩放、拖拽与文本选中拦截规则。

## 4. 日、周、月视图

### 4.1 视图层级

```ts
type CalendarView = 'day' | 'week' | 'month'
```

- 可见分段控件显示“日 / 周 / 月”。
- 双指收拢依次切换 `day -> week -> month`。
- 双指展开依次切换 `month -> week -> day`。
- 每次连续手势最多切换一级。
- 日视图左右导航一天，周视图导航一周，月视图导航一个月。

### 4.2 月视图

- 展示 `selectedDate` 所在月份。
- 网格按周一开头，固定显示六行七列，保证跨月翻页时布局高度稳定。
- 月初和月末使用相邻月份日期补齐网格，不显示空白格。
- 相邻月份日期使用较浅文字和边线，但保留完整状态标记。
- 点击当前月或相邻月日期，都原子更新 `selectedDate`、切换到日视图并打开对应日计划页。
- 今天、选中日、已完成、未完成、待处理、未来和无计划状态继续使用现有单日概览派生。

## 5. 持久化迁移

计划 store 版本从 v2 升至 v3：

- v1 数据继续补充默认 `calendarView: 'day'`。
- v2 数据中的 `day` 和 `week` 原样保留。
- v2 数据中的 `year` 迁移为 `month`。
- v3 非法视图值回退为 `day`。
- `selectedDate` 和全部 `records` 原样保留。

备份数据契约继续引用 `CalendarView`，因此类型同步变为日、周、月；本次仍不实现实际导入导出。

## 6. 柔软纸页翻动

### 6.1 运动模型

移除整张页面从 0° 到 180° 的刚性旋转，改为四层协同运动：

1. 当前页随手指横向移动，使用缓出进度减少末段突兀感。
2. 当前页只产生最高约 10° 的轻微 Y 轴倾斜，方向随翻页方向变化。
3. 目标页从约 6% 的反向偏移位置轻微视差进入。
4. 翻动边缘叠加窄幅弯曲带、高光和软阴影，强度在中段最高、首尾归零。

页面内容始终保持可读方向，不再出现 90°附近整页消失或背面反转。

### 6.2 跟手与释放

- 横向拖动进度按页面宽度计算并限制在 0 到 1。
- 视觉进度使用平滑曲线，手指位置仍保持直接、可预测的跟随关系。
- 松手时继续使用距离和速度共同判断完成或回弹。
- 完成翻页使用低阻尼弹簧推进到整页离场；回弹使用略高阻尼弹簧回到原位。
- 顶部前后按钮从静止状态使用同一完成弹簧，不建立另一套动画。
- 动画令牌继续防止被缩放取消的旧动画提交日期变化。

### 6.3 光影与纸张边缘

- 当前页移动方向的内侧边缘显示柔和投影，宽度随进度略微扩大。
- 翻动边缘使用一条纸色高光和一条低透明度阴影形成轻微卷曲感。
- 目标页只做轻微明暗过渡，不整页闪白或突然显露。
- 所有阴影透明度在开始和结束时为零，在中段达到峰值。
- 保留宣纸底色和横线纹理，不增加拟真噪点或重纹理。

### 6.4 手势边界

- 输入、按钮、链接和禁止翻页区域仍不启动翻页。
- 双指缩放优先于单指翻页；第二根手指出现时立即弹回并移交控制权。
- 横向意图成立前不阻止纵向滚动。
- 翻页完成前拒绝第二次范围导航。
- 减少动态效果模式下直接提交范围变化并短暂淡入。

## 7. 组件调整

- 将 `YearView` 替换为单月 `MonthView`。
- 将年份日期工具替换为按月导航和固定 42 格月历生成工具。
- `ViewSwitcher` 将“年”替换为“月”。
- `DateNavigator` 在月视图显示当前年月并使用“上个月 / 下个月”名称。
- `PageTurn` 保留现有公共接口，内部从旋转模型改为平移、轻倾斜、视差和边缘光影模型。
- 页面组合、store action 和点击日期进入日页的调用边界保持不变。

## 8. 错误与降级

- v2 `year` 迁移时始终映射到 `month`，不得丢失当前日期。
- 月历生成遇到跨年月份时正常补齐上一年或下一年日期。
- 手势取消、窗口失焦和组件卸载继续复位动画与锁定状态。
- 不支持 3D transform 或开启减少动态效果时，使用横向淡入替代轻倾斜和边缘卷曲。

## 9. 测试

- `CalendarView` 层级只包含日、周、月。
- v2 `year` 无损迁移为 v3 `month`。
- 月导航保持合理日期；1 月和 12 月正确跨年。
- 月历固定生成 42 个日期，周一开头并包含相邻月份补位。
- 相邻月份日期显示为弱化状态但仍可点击。
- 点击任意月历日期进入对应日视图。
- 捏合方向在日、周、月之间逐级切换且不越界。
- 柔软翻页的平移、倾斜、视差和阴影进度纯函数在 0、中点和 1 时符合边界。
- 完成、回弹、按钮翻页、双指取消和减少动态效果保持原有行为。
- 完整执行类型检查、Lint、测试和生产构建。

## 10. 完成标准

- 页面不再执行整页 180°刚性旋转。
- 拖动时当前页、目标页和边缘光影连续变化，没有 90°消失或突然换页。
- 日、周、月三个视图均可正常导航和柔软翻页。
- 月视图固定六周，补位日期可见、状态正确且可以进入日页。
- 原 v2 年视图状态升级后自动进入对应日期的月视图，计划记录不丢失。
- 自动化验证全部通过，真实手机和触控板手感记录人工验收结果。
