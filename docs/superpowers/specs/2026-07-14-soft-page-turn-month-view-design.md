# Rike 柔软翻页、月视图、工具栏与统计设计

## 1. 背景

第三阶段已经实现日、周、年视图和整页 CSS 3D 翻转。实际体验中，当前纸页围绕固定边缘旋转 180°，拖动距离与旋转角度线性对应，并使用固定窄阴影，整体更像硬质门板，翻页过程生硬。

本次调整以更柔软、克制的纸页运动替换原有整页翻转，同时将日历层级从日、周、年改为日、周、月。原有年视图和十二个月总览完全移除。

本次同时补充全局工具栏、统一返回和回到今天入口，增加独立本周总结与多范围总统计页面，并提供系统、日间、夜间三态主题。

## 2. 目标

- 日历视图改为 `day / week / month`。
- 月视图一次展示一个月，并用相邻月份日期补齐完整周。
- 月视图中的所有日期都可点击并进入对应日计划页。
- 日、周、月同层级导航统一使用柔软纸页翻动。
- 保留左右跟手、完成、回弹、按钮自动翻页和减少动态效果降级。
- 从现有 v2 本地状态无损迁移到 v3。
- 提供左上角工具栏、统一返回位置和回到今天按钮。
- 日视图可以直接进入选中日期所在周的总结页面。
- 总统计支持本周、本月、本年和全部四种范围，统计口径截至今天。
- 支持系统、日间、夜间主题选择并持久化。

## 3. 非目标

- 不保留年视图入口或十二个月总览。
- 不使用 Canvas、WebGL 或物理网格模拟真正的纸张弯曲。
- 不改变计划模板、编辑权限和单项状态派生规则。
- 不改变原生缩放、拖拽与文本选中拦截规则。

## 4. 日、周、月视图

### 4.1 视图层级

```ts
type CalendarView = 'day' | 'week' | 'month'
```

- 可见分段控件显示“日 / 周 / 月”。
- 双指收拢依次切换 `day -> week -> month`。
- 双指展开依次切换 `month -> week -> day`。
- 每次连续手势最多切换一级。
- 日视图左右导航一天，周视图导航一周，月视图导航一个月。

### 4.2 月视图

- 展示 `selectedDate` 所在月份。
- 网格按周一开头，固定显示六行七列，保证跨月翻页时布局高度稳定。
- 月初和月末使用相邻月份日期补齐网格，不显示空白格。
- 相邻月份日期使用较浅文字和边线，但保留完整状态标记。
- 点击当前月或相邻月日期，都原子更新 `selectedDate`、切换到日视图并打开对应日计划页。
- 今天、选中日、已完成、未完成、待处理、未来和无计划状态继续使用现有单日概览派生。

## 5. 持久化迁移

计划 store 版本从 v2 升至 v3：

- v1 数据继续补充默认 `calendarView: 'day'`。
- v2 数据中的 `day` 和 `week` 原样保留。
- v2 数据中的 `year` 迁移为 `month`。
- v3 非法视图值回退为 `day`。
- `selectedDate` 和全部 `records` 原样保留。

备份数据契约继续引用 `CalendarView`，因此类型同步变为日、周、月；本次仍不实现实际导入导出。

## 6. 柔软纸页翻动

### 6.1 运动模型

移除整张页面从 0° 到 180° 的刚性旋转，改为四层协同运动：

1. 当前页随手指横向移动，使用缓出进度减少末段突兀感。
2. 当前页只产生最高约 10° 的轻微 Y 轴倾斜，方向随翻页方向变化。
3. 目标页从约 6% 的反向偏移位置轻微视差进入。
4. 翻动边缘叠加窄幅弯曲带、高光和软阴影，强度在中段最高、首尾归零。

页面内容始终保持可读方向，不再出现 90°附近整页消失或背面反转。

### 6.2 跟手与释放

- 横向拖动进度按页面宽度计算并限制在 0 到 1。
- 视觉进度使用平滑曲线，手指位置仍保持直接、可预测的跟随关系。
- 松手时继续使用距离和速度共同判断完成或回弹。
- 完成翻页使用低阻尼弹簧推进到整页离场；回弹使用略高阻尼弹簧回到原位。
- 顶部前后按钮从静止状态使用同一完成弹簧，不建立另一套动画。
- 动画令牌继续防止被缩放取消的旧动画提交日期变化。

### 6.3 光影与纸张边缘

- 当前页移动方向的内侧边缘显示柔和投影，宽度随进度略微扩大。
- 翻动边缘使用一条纸色高光和一条低透明度阴影形成轻微卷曲感。
- 目标页只做轻微明暗过渡，不整页闪白或突然显露。
- 所有阴影透明度在开始和结束时为零，在中段达到峰值。
- 保留宣纸底色和横线纹理，不增加拟真噪点或重纹理。

### 6.4 手势边界

- 输入、按钮、链接和禁止翻页区域仍不启动翻页。
- 双指缩放优先于单指翻页；第二根手指出现时立即弹回并移交控制权。
- 横向意图成立前不阻止纵向滚动。
- 翻页完成前拒绝第二次范围导航。
- 减少动态效果模式下直接提交范围变化并短暂淡入。

## 7. 组件调整

- 将 `YearView` 替换为单月 `MonthView`。
- 将年份日期工具替换为按月导航和固定 42 格月历生成工具。
- `ViewSwitcher` 将“年”替换为“月”。
- `DateNavigator` 在月视图显示当前年月并使用“上个月 / 下个月”名称。
- `PageTurn` 保留现有公共接口，内部从旋转模型改为平移、轻倾斜、视差和边缘光影模型。
- 页面组合、store action 和点击日期进入日页的调用边界保持不变。

## 8. 错误与降级

- v2 `year` 迁移时始终映射到 `month`，不得丢失当前日期。
- 月历生成遇到跨年月份时正常补齐上一年或下一年日期。
- 手势取消、窗口失焦和组件卸载继续复位动画与锁定状态。
- 不支持 3D transform 或开启减少动态效果时，使用横向淡入替代轻倾斜和边缘卷曲。

## 9. 测试

- `CalendarView` 层级只包含日、周、月。
- v2 `year` 无损迁移为 v3 `month`。
- 月导航保持合理日期；1 月和 12 月正确跨年。
- 月历固定生成 42 个日期，周一开头并包含相邻月份补位。
- 相邻月份日期显示为弱化状态但仍可点击。
- 点击任意月历日期进入对应日视图。
- 捏合方向在日、周、月之间逐级切换且不越界。
- 柔软翻页的平移、倾斜、视差和阴影进度纯函数在 0、中点和 1 时符合边界。
- 完成、回弹、按钮翻页、双指取消和减少动态效果保持原有行为。
- 完整执行类型检查、Lint、测试和生产构建。

## 10. 翻页与月视图完成标准

- 页面不再执行整页 180°刚性旋转。
- 拖动时当前页、目标页和边缘光影连续变化，没有 90°消失或突然换页。
- 日、周、月三个视图均可正常导航和柔软翻页。
- 月视图固定六周，补位日期可见、状态正确且可以进入日页。
- 原 v2 年视图状态升级后自动进入对应日期的月视图，计划记录不丢失。
- 自动化验证全部通过，真实手机和触控板手感记录人工验收结果。

## 11. 工作区导航与工具栏

### 11.1 页面模式

日历视图与功能页面分开建模：

```ts
type CalendarView = 'day' | 'week' | 'month'
type WorkspacePage = 'calendar' | 'week-summary' | 'statistics'
```

- `CalendarView` 只描述日历层级。
- `WorkspacePage` 描述当前显示日历、本周总结或总统计。
- 工作区页面模式和返回快照属于会话状态，不进入计划持久化数据。
- 日期、日历视图和主题偏好继续分别持久化。

### 11.2 左上角统一槽位

- 普通日历页面左上角显示工具栏按钮。
- 从工具栏或日视图快捷入口进入子页面后，同一位置替换为返回按钮。
- 返回快照保存进入前的日期、日历视图和页面模式。
- 返回时完整恢复快照，不强制跳到今天或日视图。
- 工具栏中的周视图和月视图会建立返回快照；日历分段控件的普通视图切换不建立快照。

### 11.3 工具栏浮层

工具栏使用直角纸面浮层，上半部分依次提供：

1. 本周总结
2. 周视图
3. 月视图
4. 总统计

下方使用分隔线隔开系统、日间、夜间主题切换。

- 点击浮层外部、按 `Escape` 或完成导航后关闭浮层。
- 打开时焦点进入第一个功能项；关闭后焦点回到工具栏按钮。
- 浮层使用菜单或对话框语义，键盘可以依次访问所有功能和主题控件。
- 日视图在视图切换控件下方额外提供“查看本周总结”按钮，与工具栏入口进入同一页面。

### 11.4 回到今天

- 日期或范围标题区域提供固定位置的“回到今天”按钮。
- 当前锚点已经属于今天时隐藏按钮。
- 日视图打开今天；周视图定位到本周；月视图定位到本月。
- 本周总结切换为今天所在周的总结。
- 总统计保持当前统计范围，只将周、月或年锚点切回包含今天的范围。
- 全部统计没有时间锚点，不显示回到今天按钮。

## 12. 本周总结与总统计

### 12.1 本周总结

- 本周总结是独立工作区页面，不要求选中日期为周日。
- 统计 `selectedDate` 所在自然周，周一开始。
- 从任意历史日期进入时显示该日期所在周；回到今天后显示本周。
- 日视图快捷按钮和工具栏“本周总结”入口使用相同行为。
- 页面使用左上角统一返回按钮恢复进入前上下文。

### 12.2 总统计范围

```ts
type StatisticsRange = 'week' | 'month' | 'year' | 'all'
```

- 页面提供“本周 / 本月 / 本年 / 全部”分段控件。
- 周、月、年范围使用 `selectedDate` 作为锚点，并支持上一范围、下一范围和回到今天。
- 全部范围不显示前后范围按钮或回到今天按钮。
- 切换统计范围不改变原日历视图返回快照。

### 12.3 统一指标

本周总结和总统计复用同一个范围统计选择器，展示：

- 计划总数
- 已完成数
- 未完成数
- 完成率
- 计划文字数
- 日结文字数
- 六科未完成分布

完成率为 `已完成数 / (已完成数 + 未完成数)`；分母为零时显示 0%，不把未到期或无有效计划日期计入分母。

### 12.4 截至今天口径

- 所有统计最多计算到本地今天。
- 未来日期和未来计划不进入总数、完成率、文字数或科目分布。
- 周六有效自由计划进入总数、完成数、未完成数和完成率，但不进入六科分布。
- 周日不产生计划项。
- 固定模板文字仍不进入计划文字数。
- “全部”从不晚于今天的最早一条本地计划记录开始统计到今天；没有历史或当天记录时只统计今天，未来记录不改变起点。
- 统计结果由纯函数实时派生，不进入 store 或 localForage。

### 12.5 范围导航

- 周范围每次移动一周，月范围每次移动一个月，年范围每次移动一年。
- 下一范围允许浏览未来范围，但统计结果仍在今天截断，因此纯未来范围显示空统计。
- 返回日历时恢复进入统计前的日期和视图，不使用统计页内改变后的锚点覆盖返回快照。

## 13. 三态主题

### 13.1 偏好与生效主题

```ts
type ThemePreference = 'system' | 'light' | 'dark'
type ResolvedTheme = 'light' | 'dark'
```

- 默认偏好为 `system`。
- `system` 实时监听 `prefers-color-scheme`，系统变化时立即更新生效主题。
- 显式选择日间或夜间后不再跟随系统，直到用户重新选择系统。
- 主题偏好独立持久化，不进入计划 store 版本迁移。
- 在 React 首次渲染前读取并应用偏好，避免刷新时出现明显主题闪烁。

### 13.2 主题入口

- 工具栏功能项下方放置“系统 / 日间 / 夜间”三段控件。
- 控件暴露当前偏好状态。
- 系统选项同时说明当前实际生效的是日间或夜间。
- 切换后立即应用并持久化，无需关闭工具栏。

### 13.3 夜间视觉

- 工作台使用近黑墨灰，纸面使用深灰褐色，不使用纯黑。
- 正文使用柔和米白，次要文字、横线和边框降低对比。
- 朱砂红、青绿和黄色警示降低亮度与饱和度，但仍满足状态区分。
- 输入、工具栏浮层、底部导航、统计图、纸页背面、翻页高光和阴影全部使用语义主题变量。
- 组件不得分别判断夜间模式，颜色切换统一由根节点主题属性和 CSS 变量完成。

### 13.4 主题错误边界

- 无法读取持久化偏好时回退到 `system`。
- 非法偏好值回退到 `system`。
- 浏览器不支持 `matchMedia` 时，系统模式回退到日间。
- 主题持久化失败不阻止当次会话切换。

## 14. 新增测试与完成标准

除第 9 节测试外，补充覆盖：

- 工具栏打开、外部关闭、Escape 关闭和焦点恢复。
- 工具栏四个功能入口与日视图本周总结快捷入口。
- 左上角返回快照恢复日期、视图和页面模式。
- 日、周、月、本周总结和统计范围的回到今天行为。
- 本周、月、年、全部统计范围与截至今天截断。
- 未来计划不进入统计，周六计划不进入科目分布。
- 没有本地记录时全部范围只统计今天。
- 系统、日间、夜间偏好恢复与非法值回退。
- 系统主题实时变化与显式主题覆盖。
- 根节点主题属性、工具栏主题控件和夜间语义颜色。

新增完成标准：

- 所有工具栏目标可以进入，并能从统一左上角位置返回原上下文。
- 日视图可以一键打开选中日期所在周的总结。
- 回到今天在所有有时间锚点的页面行为一致。
- 总统计四个范围使用截至今天的统一口径。
- 系统、日间、夜间主题可切换、可恢复且无明显首屏闪烁。
