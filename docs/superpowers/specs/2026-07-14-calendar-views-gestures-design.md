# Rike 第三阶段：日周年视图、缩放手势与翻页设计

## 1. 背景

Rike 已完成日计划模板、周六自由计划、周日统计、日期切换和本地持久化。第三阶段在现有计划数据之上建立完整的日、周、年三个日历层级，并加入可见视图切换、双指或触控板缩放、同层级范围导航和真实纸页翻动。

本阶段不修改既有计划模板、编辑权限和完成状态规则。周视图与年视图只派生现有记录，不建立重复的摘要数据。

## 2. 目标

- 日视图继续提供现有计划编辑能力。
- 周视图展示周一至周日七天摘要，并允许选择日期。
- 年视图展示十二个月日期网格和计划状态，并允许选择日期。
- 提供始终可见的“日 / 周 / 年”分段切换控件。
- 支持手机双指和电脑触控板捏合，在三个视图层级间逐级切换。
- 日、周、年视图在同层级左右导航时都使用可跟手的 CSS 3D 纸页翻动。
- 强制拦截应用可控制的浏览器原生缩放、元素拖拽和非编辑文本选中。
- 保留现有 Zustand、localForage、日期权限和恢复失败写保护机制。

## 3. 非目标

- 不实现 Canvas 或 WebGL 纸张弯曲模拟。
- 不增加模板配置或周期预设管理。
- 不实现数据导入导出、正式每日一句内容库或完整 PWA 离线能力。
- 不改变工作日、周六和周日现有业务规则。
- 不尝试拦截网页无法控制的浏览器菜单缩放命令。

## 4. 总体方案

采用“离散三视图 + 共享日期游标”方案：

- `DayView`、`WeekView` 和 `YearView` 分别负责各自布局。
- `selectedDate` 和 `calendarView` 共同构成日历游标。
- `CalendarWorkspace` 组合范围导航、视图切换、手势识别和动画。
- 周、年概览通过纯选择器从计划记录实时派生。
- 视图层级切换使用轻量缩放动画；同层级日期范围导航使用 CSS 3D 翻页动画。

不采用单一连续变形画布，避免复杂布局测量和移动端性能风险；不将三个视图拆成路由，避免日期上下文和缩放连续感割裂。

## 5. 状态与持久化

### 5.1 日历游标

计划 store 增加：

```ts
type CalendarView = 'day' | 'week' | 'year'

interface CalendarCursor {
  selectedDate: string
  calendarView: CalendarView
}
```

- 切换视图不改变 `selectedDate`。
- 点击周或年视图中的日期只更新 `selectedDate`，保持当前视图。
- 用户通过分段控件或放大手势进入日视图后，展示当前选中日期。
- `selectedDate` 和 `calendarView` 都进入持久化状态，刷新后恢复。

### 5.2 版本迁移

持久化版本从 v1 升至 v2，并提供显式迁移：

- 保留 v1 的 `selectedDate` 和全部 `records`。
- 为 v1 数据补充 `calendarView: 'day'`。
- v2 数据中的非法视图值回退为 `day`。
- 迁移或恢复失败时沿用现有写保护，不能用默认数据覆盖已有本地记录。

## 6. 三个视图

### 6.1 公共导航

- 顶部保留日期或范围标题、上一范围按钮和下一范围按钮。
- 标题下方提供直角线框式“日 / 周 / 年”分段控件。
- 前后按钮的单位与无障碍名称随视图变化：前一天/后一天、上一周/下一周、上一年/下一年。
- 每日一句继续跟随 `selectedDate`，三个视图中均可显示。

### 6.2 日视图

- 复用现有工作日计划、周六自由列表和周日总结。
- 左右翻页每次移动一天。
- 原有计划输入、状态切换、日期权限和统计规则保持不变。

### 6.3 周视图

- 展示包含 `selectedDate` 的自然周，周一为起始日。
- 使用七个日期摘要块展示日期、计划总数、完成数、未完成数和整体状态。
- 选中日、今天、完成状态、未完成警报、待处理和未来状态需要通过线条、文字和颜色共同区分，不能只依赖颜色。
- 周日摘要块额外展示整周完成比例。
- 周六没有有效条目时显示为空状态，不制造计划或未完成警报。
- 点击日期只选择该日，不自动进入日视图。
- 左右翻页每次移动一周，并尽量保持选中日期的星期位置。

### 6.4 年视图

- 展示 `selectedDate` 所在年份的十二个月网格。
- 每个月按周一开头排列日期格，日期格显示日号和整体状态标记。
- 今天、选中日、全部完成、存在未完成、当天待处理、未来日期和无有效计划均有明确且互不冲突的表现。
- 无有效计划的日期不显示完成或未完成标记。
- 点击日期只选择该日，不自动进入日视图。
- 左右翻页每次移动一年，并尽量保持选中日期的月和日；闰日进入非闰年时使用该月最后一个有效日期。

## 7. 状态派生

新增纯函数生成单日概览：

```ts
interface DayOverview {
  totalPlans: number
  completedPlans: number
  missedPlans: number
  status: 'empty' | 'upcoming' | 'pending' | 'completed' | 'missed'
}
```

派生规则：

- 只统计既有规则认定的有效计划。
- 存在一个或多个未完成项时，日期整体状态为 `missed`。
- 有有效计划且全部完成时为 `completed`。
- 当天有未决的有效计划时为 `pending`。
- 未来有有效计划时为 `upcoming`。
- 没有有效计划时为 `empty`。
- 周六空白行不是有效计划。
- 周日自身不产生计划项，周视图中的周日额外读取整周统计。

周视图组合七个 `DayOverview` 与现有 `WeekSummary`。年视图为当前年份的日期生成 `DayOverview`。概览不进入 store，也不写入 localForage。

## 8. 视图缩放

### 8.1 层级规则

- 双指收拢：`day -> week -> year`。
- 双指展开：`year -> week -> day`。
- 触控板捏合方向与手机一致。
- 每次连续手势最多跨越一个层级；手势结束后才允许下一次切换。
- 在最细或最粗层级继续手势不会循环或改变日期。
- 进入更细视图时围绕 `selectedDate` 展开；进入更粗视图时保持该日期高亮并置于可见区域。

### 8.2 手机手势

- 双指开始时记录初始距离。
- 距离变化跨过阈值后切换一级并锁定本次手势。
- 第二根手指出现后立即取消正在进行的单指翻页。
- 双指 `touchmove` 使用非被动监听器并阻止浏览器默认缩放。
- 单指纵向滚动、表单输入和点击不被缩放识别器接管。

### 8.3 PC 手势

- 带 `ctrlKey` 的 `wheel` 事件视为浏览器或触控板缩放输入。
- 在日历工作区内累计触控板缩放量，跨过阈值后切换一级并锁定，待手势静默结束后复位。
- 普通鼠标滚轮只负责页面滚动，不改变视图。

### 8.4 可见控件与无障碍

- 手势不是唯一入口，分段控件始终可以直接选择日、周、年视图。
- 控件使用真实按钮语义并暴露当前选中状态。
- 开启“减少动态效果”时，视图立即切换，不依赖动画传达层级变化。

## 9. 强制禁止原生缩放、拖拽与选中

### 9.1 原生缩放

- viewport 设置 `maximum-scale=1, user-scalable=no`，禁止常规手机页面缩放。
- 全局拦截 Safari 的 `gesturestart`、`gesturechange` 和 `gestureend`。
- 全局阻止多指 `touchmove` 的默认缩放；日历工作区仍读取触点距离用于应用视图切换。
- 全局拦截 `Ctrl/Command + wheel`。日历工作区将其转换为应用视图缩放，其他区域不执行缩放。
- 全局拦截 `Ctrl/Command + +`、`-`、`=` 和 `0` 缩放快捷键。
- 浏览器菜单触发的页面缩放无法由网页代码可靠拦截，不列入完成标准。

需要阻止默认行为的监听器显式使用 `{ passive: false }`，并在应用卸载时清理。

### 9.2 拖拽与文本选中

- 全局阻止浏览器原生 `dragstart`，避免文字、链接和图标产生拖拽幽灵图。
- 页面非编辑区域统一使用 `user-select: none`。
- `input`、`textarea` 和其他明确可编辑区域恢复 `user-select: text`，保留选中、复制、粘贴和选择手柄。
- 应用自有的翻页、左右手势与缩放不属于原生元素拖拽，继续可用。

## 10. CSS 3D 翻页

### 10.1 范围

- 日视图跨日、周视图跨周、年视图跨年均使用同一翻页容器。
- 应用导航栏固定，只翻动内部计划纸页。
- 日、周、年之间的层级切换不使用翻页，而使用缩放和淡入淡出。

### 10.2 跟手交互

- 单指从非交互区域横向拖动时，当前纸页围绕对应侧边实时旋转。
- 向左翻到下一范围，向右翻到上一范围。
- 翻转进度同时驱动透视角度、纸页背面亮度、边缘阴影和下一页显露程度。
- 松手时根据横向距离和速度决定完成翻页或弹回原页。
- 顶部前后按钮触发同方向的自动翻页。

### 10.3 页面与交互约束

- 翻页期间同时渲染旧页和目标页，但只有当前可见页允许交互。
- 范围状态在动画完成后提交，避免中途改变输入组件与可访问焦点。
- 输入框、文本域、按钮、链接和标记为禁止翻页的区域不能启动翻页。
- 翻页进行中不接受第二次范围导航。
- 双指一旦出现，取消并复位单指翻页，交给视图缩放。
- 使用稳定透视、`transform-style: preserve-3d` 和隐藏背面；背面沿用宣纸底色与线条纹理。
- 开启“减少动态效果”时不执行 3D 旋转，范围直接更新并短暂淡入。

## 11. 手势优先级

手势冲突按以下顺序处理：

1. 可编辑控件自身的输入、光标和选择行为。
2. 双指或触控板捏合的视图缩放。
3. 非交互区域的单指横向翻页。
4. 单指纵向页面滚动。

一旦双指缩放获得控制权，本次手势内不得再次启动翻页。横向翻页未达到意图阈值时应允许纵向滚动，避免页面轻微横移就锁死滚动。

## 12. 动画与性能

- 使用 Motion 驱动离散状态与弹簧回弹，CSS 3D transform 驱动翻页视觉。
- 翻页只保留当前页和相邻目标页，动画结束后立即卸载旧页。
- 年视图最多渲染当前年份十二个月，不预渲染多个完整年份。
- 概览计算保持为纯函数，并在页面层按 `records`、年份或周范围进行记忆化。
- 动画只改变 transform、opacity 和必要的阴影参数，避免持续触发布局计算。

## 13. 错误与降级

- 无法识别的持久化视图值回退到日视图。
- 持久化恢复失败时继续显示现有警告并阻止默认状态回写。
- 不支持 Safari gesture 事件的浏览器只使用标准触摸和 wheel 监听器。
- 不支持 3D 背面表现或开启减少动态效果时，降级为即时范围切换与淡入。
- 手势取消、窗口失焦或组件卸载时必须清理锁定状态，不能让页面永久失去滚动或导航能力。

## 14. 测试

### 14.1 纯函数与状态

- 单日概览的空、未来、待处理、完成和未完成状态。
- 周一开头的周日期生成和跨年边界。
- 十二个月网格、闰年和闰日跨年导航。
- 日、周、年导航单位。
- 缩放层级边界、阈值、方向和一次手势只切换一级。
- 翻页距离、速度阈值与完成或回弹判定。
- v1 到 v2 迁移、非法视图回退和记录保留。

### 14.2 组件与集成

- 分段控件可切换视图并暴露选中状态。
- 周视图七日摘要、点击选日与周日完成比例。
- 年视图十二个月、状态标记和点击选日。
- 三个视图的前后按钮使用正确单位。
- 输入区域不会启动翻页，双指会取消单指翻页。
- 翻页期间目标页不可交互，完成后焦点与内容正确。
- 原生缩放快捷键、`ctrlKey` wheel、多指 touch 和 dragstart 被阻止。
- 非编辑内容不可选择，可编辑内容仍允许文本选择。
- 减少动态效果下不依赖 3D 动画。

### 14.3 阶段验证

阶段完成时执行：

```bash
pnpm typecheck
pnpm lint
pnpm test
pnpm build
```

真实手机双指、电脑触控板、Safari 手势事件、跟手翻页手感和浏览器缩放拦截需要人工验收。当前环境缺少对应设备或浏览器时，交接文档必须明确记录未执行项。

## 15. 完成标准

- 三个视图均可通过可见控件进入并正确展示当前选中日期范围。
- 周视图和年视图的状态与现有计划记录一致，不保存重复摘要。
- 手机双指和电脑触控板捏合可以逐级切换视图，单次手势不会跨越多级。
- 三个视图内的左右范围导航都具有可跟手、可完成、可回弹的纸页翻动。
- 应用可控制的手机、触控板和快捷键原生缩放被强制阻止。
- 浏览器原生元素拖拽和非编辑文本选中被阻止，可编辑文本仍能正常选中和修改。
- v1 本地计划数据升级后完整保留。
- 类型检查、Lint、测试和生产构建全部通过。
- 第三阶段交接文档记录实现路径、验证结果、人工验收项和后续事项。
