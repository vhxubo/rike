# Rike 需求过程记录

## 记录说明

本文记录需求在讨论中的新增、确认和阶段划分。具体技术设计以 `docs/superpowers/specs/2026-07-14-rike-foundation-design.md` 为准。

## 2026-07-14 初始需求

项目名称为 Rike，中文名“日课”。目标是开发一个线条风格的每周学习计划应用。

指定技术栈：

- React
- Vite
- Tailwind CSS
- Zustand
- TypeScript
- localForage
- Motion
- Tailwind CSS Variants

总体能力要求：

- 支持每周学习计划。
- 支持预设模板，并能为不同周期设置不同预设。
- 支持完整动画，包括翻页效果和缩放手势效果。
- 优先支持手机端，其次支持 PC 端。
- 支持 PWA。
- 每次操作后保存到本地，刷新后恢复。
- 第一阶段先完成所需组件的项目搭建。
- 每次开发后留下交接文档。

## 2026-07-14 第一阶段范围确认

讨论了三个范围：基础骨架、可运行底座和首屏原型。最终确认使用“基础骨架”：

- 初始化工程与工具链。
- 建立应用分层和组件目录。
- 完成主题变量、基础 UI、布局与组件陈列页。
- PWA、持久化和真实计划业务只预留边界，不在本阶段实现。

## 2026-07-14 视觉方向确认

线条风格选择“纸本日课”，而不是理性网格或极简书页。

设计意图：

- 使用横线纸、装订线和墨色描边。
- 使用克制的朱砂红作为重点色。
- 保持现代、清晰，不做重度仿旧。
- 组件陈列页作为本轮首页和验收入口。

## 2026-07-14 架构确认

确认采用分层应用骨架：

- `app` 管理应用入口和全局能力。
- `pages` 负责页面组合。
- `components` 按 UI、布局、纸本和反馈分类。
- `features` 隔离业务领域。
- `stores` 和 `lib/storage` 为后续状态与存储提供边界。

通用组件保持 props 驱动，不直接读取 Zustand 或 localForage。

## 2026-07-14 基础组件确认

确认第一阶段建立以下组件类别：

- 按钮、图标按钮、输入、文本域、复选框、开关、标签、分隔线和分段控制器。
- 纸页、横线区域、装订线和日期印记。
- 应用壳、顶部栏、底部导航和内容容器。
- 空状态和行内提示。

Motion 在本阶段仅用于轻量入场和按压反馈。翻页和缩放手势留到业务阶段。

## 2026-07-14 新增总体需求

新增以下能力：

- 提供数据导入和导出，方便备份与迁移。
- 首页展示日期。
- 左右滑动切换不同日期。
- 手指缩放实现日、周、年视图切换。
- 头部展示每日一句。
- 每日一句使用约 400 条本地数据。

每日一句的用户背景涉及未成年学生、语言能力恢复和严重心理健康经历。内容方向确认为：

- 与学习相关，适合初二学生。
- 使用短句和具体、可完成的引导。
- 肯定努力与进步，但不夸大效果。
- 不诊断或强化未经专业评估的人格状态。
- 不反复提示创伤、疾病或身份标签。
- 不使用羞辱、威胁、内疚或依赖性表达。

## 2026-07-14 新增需求的阶段归属

确认继续执行基础搭建范围。新增功能本轮只记录并预留模块：

- `features/calendar`：日期游标、日/周/年视图与手势接口。
- `features/backup`：带版本号的 JSON 备份契约。
- `features/daily-guidance`：每日一句数据结构和筛选边界。

第一阶段首页会展示当前日期和一条示例学习引导语，但不实现日期滑动、缩放切换、实际导入导出或 400 条正式内容。

## 后续建议顺序

1. 建立周计划领域模型与 Zustand 状态层。
2. 实现日期游标、左右滑动和日/周/年缩放切换。
3. 接入 localForage，并实现刷新恢复。
4. 实现带校验和版本迁移的数据导入导出。
5. 生成、分类并校审 400 条每日一句本地数据。
6. 实现 PWA、离线缓存与安装体验。
7. 完成翻页动画与跨视图动效打磨。

## 2026-07-14 第二阶段：日期模板与周统计

新增周一至周五、周六和周日三类日期页面。完整设计见 `docs/superpowers/specs/2026-07-14-date-templates-design.md`。

讨论中确认的关键规则：

- 周一/三/五使用语文、化学主模板；周二/四使用英语、数学主模板。
- 第 2、5 项的“错题/知识点”是完整固定文案，不是二选一。
- 只有工作日第 4、6、7 项指定输入区域可编辑。
- 模板项必须包含明确 `subject` 字段，后续统计不根据文字猜科目。
- 周六使用结构化自动编号列表，每行独立状态，整体使用工作日 `1.5x` 视觉尺寸。
- 周日统计计划输入字数、日结字数、计划总数、完成数、未完成数和六科未完成分布。
- 工作日第 6 项和周六条目计入计划数量，但不进入六科分布。
- 状态只允许在当天调整；过去状态与内容只读，未来可提前填写计划但不能调整状态或填写日结。
- 当天状态循环为 `待处理 -> 已完成 <-> 未完成`；未完成使用与过去未完成相同的红底和黄色警报样式。
- 固定模板文字不计入计划字数，字数去除空格和换行后计算，标点计入。
- 本阶段采用完整功能切片，接入 Zustand + localForage，并实现日期按钮和左右滑动；缩放与翻页留到后续。

## 2026-07-14 第三阶段：日周年视图、缩放与翻页

第三阶段选择完整可用的日、周、年视图，而不是仅用于导航的缩略视图或动效占位。完整设计见 `docs/superpowers/specs/2026-07-14-calendar-views-gestures-design.md`。

讨论中确认的关键规则：

- 周视图展示七天计划摘要并可选日，年视图展示十二个月日期网格和计划状态。
- 点击周、年视图中的日期会选择该日期，并立即进入对应的日计划页。
- 保留可见的日、周、年切换控件，缩放手势不是唯一入口。
- 手机双指和电脑触控板捏合逐级切换视图，每次手势最多切换一级。
- 强制拦截应用可控制的手机、触控板和快捷键原生缩放。
- 日视图左右移动一天、周视图移动一周、年视图移动一年。
- 三个视图的同层级左右导航均采用可跟手的 CSS 3D 真实纸页翻动。
- 双指缩放优先于单指翻页，出现第二根手指时取消翻页。
- 全局阻止原生元素拖拽和非编辑文本选中；输入框、文本域等可编辑区域仍允许选中和修改。
- 日历视图状态进入版本化持久化，并从现有 v1 数据无损迁移到 v2。

## 2026-07-14 翻页手感与视图层级调整

第三阶段实际体验后确认原有整页 180°旋转过于生硬，后续调整为柔软纸页运动，并将视图层级从日、周、年改为日、周、月。完整设计见 `docs/superpowers/specs/2026-07-14-soft-page-turn-month-view-design.md`。

确认规则：

- 原有年视图和十二个月总览全部移除。
- 月视图固定展示六周七列，并显示相邻月份补位日期。
- 补位日期弱化显示但可以点击，点击后进入对应日计划页。
- 日、周、月同层级导航使用平移、轻倾斜、视差和边缘光影组合的柔软纸页翻动。
- store 从 v2 升到 v3，原 `year` 视图迁移为 `month`，日期和计划记录保持不变。
- 左上角增加可展开工具栏，提供本周总结、周视图、月视图和总统计入口。
- 工具栏目标使用同一左上角位置返回，并恢复进入前的日期与视图。
- 日期和范围标题区域增加回到今天按钮。
- 日视图增加直接进入选中日期所在周总结的按钮。
- 总统计支持本周、本月、本年和全部范围，所有统计截至今天。
- 工具栏下方提供系统、日间、夜间三态主题切换，系统模式实时跟随系统主题。

## 2026-07-14 页面壳与翻页细化

完整设计见 `docs/superpowers/specs/2026-07-14-sticky-header-dynamic-fold-fab-design.md`。

确认规则：

- header 吸附在视口顶部，位于翻页容器外，不随内容滚动或日期翻页。
- 右上角放本周总结和日/周/月切换，替代设置图标与内容区切换控件。
- 移除底部 TabBar、`TODAY` 和 `PLAN` 展示文字。
- 每日一句放到原 `TODAY` 首行，日期导航和计划内容依次下移。
- 翻页纸张与阴影不被直接父级 overflow 裁切。
- 折痕中心和角度跟随手指纵向位置，按钮翻页默认从 60% 高度起折。
- 回到今天改为可拖动 FAB，只在当前日期或范围不包含今天时显示，并持久化归一化位置。
- 手机端取消页面外侧左右 gutter，内容内部阅读边距保留。
